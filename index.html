Mi enfoque en QA

Me enfoco en comprender a fondo los requerimientos y diseñar casos de prueba claros que detecten posibles fallos antes de que impacten al usuario final. Aplico un pensamiento analítico y criterioso, asegurando la confiabilidad funcional y de los datos de los productos digitales.

Combino testing manual con fundamentos de automatización, pruebas cross-browser, validación de APIs y revisión de flujos críticos, siempre buscando mejorar la experiencia del usuario.

Me motiva la mejora continua y trabajar de manera remota colaborando con equipos multidisciplinarios, garantizando comunicación clara, documentación organizada y un enfoque proactivo hacia la calidad.

Proyecto Urban Routes – App de Car-Sharing

Urban Routes permite a los usuarios reservar taxis fácilmente. Probé automáticamente todo el flujo de reserva: ruta, tipo de servicio, teléfono, pago y extras.

Automatización E2E con Python y Selenium WebDriver, Page Object Model (POM). Cobertura completa de escenarios críticos y logs.

×

Proyecto Urban Grocers – API Automation

Urban Grocers permite crear kits personalizados de productos. Automatizamos pruebas para asegurar que la API funcione correctamente.

Python con requests y pytest, arquitectura modular y validación de 9 casos de prueba.

×

Proyecto Urban Scooter

Probé la app móvil, web y APIs: pedidos, notificaciones, cancelaciones y conectividad offline.

Cobertura completa: UI, móvil y API. Diseño de pruebas: clases de equivalencia, valores límite y tablas de decisión. Validación de datos SQL relacional. Herramientas: Postman, emuladores, pruebas manuales UI.

Proyecto Mercado Libre – Botón “Agregar al carrito”

Durante la navegación, el botón “Agregar al carrito” se activa accidentalmente al hacer scroll, agregando productos sin intención.

Se propone mover el botón fuera del área de scroll y agregar confirmación o “Deshacer” inmediata.
