Urban Routes – App de Car-Sharing

Descripción simple: Urban Routes permite a los usuarios reservar taxis fácilmente. Probé automáticamente todo el flujo de reserva: ruta, tipo de servicio, teléfono, pago y extras.

Urban Grocers – API Automation

Descripción simple: Urban Grocers permite crear kits personalizados de productos. Automatizamos pruebas para asegurar que la API funcione correctamente y que los datos cumplan las reglas.

Detalles técnicos: Python con requests y pytest, arquitectura modular: separación de configuración, datos y lógica de envío. Validación de 9 casos de prueba con positive/negative assertions y autenticación mediante tokens.

Urban Scooter – Proyecto Final

Descripción simple: Urban Scooter permite solicitar scooters compartidos. Probé la app móvil, web y APIs: pedidos, notificaciones, cancelaciones y conectividad offline.

Detalles técnicos: Cobertura completa: UI, móvil y API. Diseño de pruebas: clases de equivalencia, valores límite y tablas de decisión. Validación de datos SQL relacional. Herramientas: Postman, emuladores móviles, pruebas manuales UI. Técnicas: pruebas positivas/negativas, exploratorias y automatizadas. Resultados: bugs críticos identificados, documentación profesional, cobertura de escenarios complejos.

Áreas de mejora: Caché, automatización estratégica de API, bases de datos NoSQL y HTTP avanzado.

Mercado Libre – Prueba UX QA

Título: Activación accidental del botón “Agregar al carrito” durante scroll

Tipo: Bug / UX Issue | Prioridad: Alta | Severidad: Alta

Descripción

Durante la navegación, el botón “Agregar al carrito” se activa accidentalmente al hacer scroll, agregando productos sin intención...

Precondiciones

  • Usuario autenticado
  • App instalada en dispositivo móvil
  • Conexión a internet activa
  • Pruebas realizadas en Android y iOS

Pasos para reproducir

  1. Abrir la app
  2. Buscar cualquier producto
  3. Ingresar a listado de resultados
  4. Hacer scroll vertical (lento o rápido)
  5. Observar interacción con los botones

Resultado actual

  • Productos se agregan sin intención
  • No hay confirmación clara ni opción de deshacer
  • Usuario debe ir al carrito para eliminar

Resultado esperado

  • Scroll no activa taps accidentalmente
  • Botón requiere interacción intencional clara
  • Opción inmediata de “Deshacer” (snackbar)
  • Usuario puede eliminar producto sin salir del listado
